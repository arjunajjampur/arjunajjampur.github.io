INSPIRE GUIDEPANELS

Overview

At Altair, we have been offering generative design to our customers since 2008.

Altair Inspire, launched in 2009 brought forward a completely new way to optimize 3D models. This was the first user experience focused Topology Optimzation software on the market. The philosophy was to place the power of generative design into the hands of anyone, persona-agnostic. Now designers could run optimizations and produce amazing results.

My contribution

I was tasked with creating a new way of capturing inputs for complex use cases. This involved both novel component design and brand new user-experience elements.

Problem

As Inspire grew in complexity, we found ourselves dealing with complicated inputs for several tools. Existing methods of capturing inputs were getting quite hairy.

The "Guide-bar" introduced with Inspire was sufficient to capture inputs for most tools but became increasingly hard to handle for newer tools that required multiple inputs. It started causing fatigue.

In some cases, the guidebar flowed across the entire width of the modeling window, forcing users to traverse this distance to complete a task.

Solution

The strategy was to build out a fully atomic design system and this involved 3 steps:

  1. Inventory existing components
  2. Build components starting from fundamental atoms
  3. Share design patterns with product teams

As a team of one, I built the fundamental components, starting with defining color palette using Leornardocolor created by Nate Baldwin.

Using figma variables, we defined global and alias tokens to ensure consistency

While building these components, we identified key product teams that were design evangelists. After setting up a formal ticketing, component request and fixing process, we rolled out the design system with a kickoff. It included an overview of the project, figma and its utility for our teams, the design system usage and guidelines.

Using the atomic approach, I built all templates to be fully self-sufficient and easily updatable. The resulting pattern library was extensive for the first launch, with over 200 components and 400 variants.

This was a brand new initiative, and we were amazed with the response. We ensured the guides were clear and everyone was empowered to contribute to this new system.

Outcome

The result of this new design system included increased sprint efficiency and velocity, higher organizational fluency, improved visibility for our team and last but not least quicker turnaround time for PMs and devs to implement powerful new features for our customers.
